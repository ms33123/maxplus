# 操作说明文档

## 1. 环境准备

本项目需要本地具备以下环境：

- Node.js：建议 `18+` 或 `20+`
- npm：随 Node.js 安装
- MySQL：建议 `8.0+`
- Git：用于版本管理

开发时需要同时启动三个服务：

| 服务 | 目录 | 默认地址 | 说明 |
| --- | --- | --- | --- |
| 后端 API | `node` | `http://localhost:4000` | Express + MySQL |
| 官网前台 | `web` | `http://localhost:5173` | Vite 开发服务 |
| 后台管理 | `sysadmin` | `http://localhost:5174` | Vite 开发服务 |

前台和后台的 `/api` 请求会代理到 `http://127.0.0.1:4000`。

## 2. 安装依赖

在项目根目录分别进入三个子项目安装依赖。

```powershell
cd F:\桌面\maxpluswebsite\node
npm install

cd F:\桌面\maxpluswebsite\web
npm install

cd F:\桌面\maxpluswebsite\sysadmin
npm install
```

如果依赖已安装，可以跳过该步骤。

## 3. 启动后端

进入后端目录：

```powershell
cd F:\桌面\maxpluswebsite\node
npm run dev
```

默认监听：

```text
http://0.0.0.0:4000
```

首次启动时，如果 `node/.env` 不存在或 `APP_INSTALLED` 未设置为 `true`，后端不会直接初始化业务数据库，而是进入“待安装”状态。此时需要启动后台管理端，访问 `/install` 完成安装。

## 4. 首次安装系统

### 4.1 启动后台管理端

打开新的 PowerShell 窗口：

```powershell
cd F:\桌面\maxpluswebsite\sysadmin
npm run dev
```

浏览器访问：

```text
http://localhost:5174/install
```

### 4.2 填写安装信息

安装页需要填写：

- 数据库主机，例如 `127.0.0.1`
- 数据库端口，例如 `3306`
- 数据库用户名，例如 `root`
- 数据库密码
- 数据库名称，例如 `maxplus_cms`
- 后台管理员账号
- 后台管理员密码

点击数据库测试成功后，提交安装。

### 4.3 安装完成后的结果

安装成功后系统会自动：

- 在 `node/.env` 写入数据库和安全配置。
- 创建数据库，不存在时自动创建。
- 创建业务数据表。
- 写入初始站点、商品、视频、博客、SEO、日志等种子数据。
- 创建后台超级管理员账号。
- 跳转到后台登录页。

安装完成后，继续保持后端运行，然后访问：

```text
http://localhost:5174/login
```

使用安装时设置的管理员账号和密码登录。

## 5. 启动官网前台

打开新的 PowerShell 窗口：

```powershell
cd F:\桌面\maxpluswebsite\web
npm run dev
```

浏览器访问：

```text
http://localhost:5173
```

前台会通过 `/api/public/bootstrap` 加载后台配置的站点内容、首页模块、商品、视频、博客和 SEO 数据。

## 6. 日常开发启动顺序

推荐顺序：

1. 启动 MySQL。
2. 进入 `node` 目录，执行 `npm run dev`。
3. 进入 `sysadmin` 目录，执行 `npm run dev`。
4. 进入 `web` 目录，执行 `npm run dev`。

常用地址：

| 地址 | 用途 |
| --- | --- |
| `http://localhost:4000/api/install/status` | 安装状态接口，只接受 POST |
| `http://localhost:5173` | 官网前台 |
| `http://localhost:5174/install` | 首次安装 |
| `http://localhost:5174/login` | 后台登录 |
| `http://localhost:5174/dashboard` | 后台仪表盘 |

注意：后端接口统一使用 `POST`，直接在浏览器地址栏打开接口一般不会得到有效结果。

## 7. 后台内容操作

### 7.1 站点设置

路径：

```text
后台 -> 系统配置 -> 站点设置
```

可维护品牌信息、社交链接、通知、页脚等基础内容。保存后，前台刷新页面即可读取新配置。

### 7.2 邮件配置

路径：

```text
后台 -> 系统配置 -> 邮件配置
```

用于配置订阅确认邮件相关参数。配置前需要确认 SMTP 服务可用。

### 7.3 商品和分类

路径：

```text
后台 -> 商品中心 -> 商品管理
后台 -> 商品中心 -> 商品分类
```

商品分类用于前台分类页和商品筛选。商品中的 `slug` 会用于前台详情页地址：

```text
/products/:slug
```

修改商品后，前台商品列表和详情页会同步变化。

### 7.4 视频内容

路径：

```text
后台 -> 视频管理 -> 视频管理
后台 -> 视频管理 -> 视频分类
```

视频 `slug` 会用于前台视频详情页地址：

```text
/videos/:slug
```

### 7.5 博客文章

路径：

```text
后台 -> 博客管理 -> 文章列表
后台 -> 博客管理 -> 文章分类
后台 -> 博客管理 -> 博客页面
```

博客文章 `slug` 会用于前台博客详情页地址：

```text
/blog/:slug
```

### 7.6 首页运营

路径：

```text
后台 -> 首页运营
```

可维护：

- 首页轮播
- 首页产品
- 首页视频
- 首页分类
- 首页评价
- 首页联系模块
- 订阅弹窗
- 首页模块开关

如果某个首页模块不需要展示，可在“首页模块开关”中关闭。

### 7.7 询盘和订阅

路径：

```text
后台 -> 线索管理 -> 询盘管理
后台 -> 线索管理 -> 订阅管理
```

前台联系表单、订阅弹窗等提交的数据会进入这些模块。敏感字段在数据库中以加密形式保存，后台通过接口读取和展示。

### 7.8 SEO

路径：

```text
后台 -> 系统配置 -> SEO
```

用于维护前台页面的 SEO 信息。前台通过 `usePageMeta`、`useStructuredData` 等组合函数应用这些配置。

### 7.9 系统日志

路径：

```text
后台 -> 系统配置 -> 系统日志
```

记录登录、内容保存、删除、询盘和订阅等操作。

## 9. 生产部署建议

### 9.1 后端部署

后端目录：

```powershell
cd node
npm install --omit=dev
npm start
```

生产环境建议明确配置 `node/.env`：

```env
APP_INSTALLED=true
HOST=0.0.0.0
PORT=4000
API_PREFIX=/api
CORS_ORIGINS=https://your-domain.com,https://admin.your-domain.com
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=maxplus_user
MYSQL_PASSWORD=your-password
MYSQL_DATABASE=maxplus_cms
JWT_SECRET=your-random-secret
ADMIN_RECOVERY_KEY=your-random-recovery-key
DATA_ENCRYPTION_KEY=your-random-data-key
JWT_EXPIRES_IN=12h
```

生产环境不要使用开发默认密钥。`CORS_ORIGINS` 建议填写具体前台和后台域名，不建议长期使用 `*`。

### 9.2 前台和后台部署

构建前如需指定后端地址，可设置：

```env
VITE_API_BASE_URL=https://your-api-domain.com/api
```

然后分别构建：

```powershell
cd web
npm run build

cd ..\sysadmin
npm run build
```

构建产物位于各自目录的 `dist/`。部署时需要配置静态服务器，并将前台和后台的前端路由都回退到对应的 `index.html`。

## 10. 常见问题

### 10.1 后台一直跳转到安装页

可能原因：

- `node/.env` 不存在。
- `APP_INSTALLED` 不是 `true`。
- 后端启动时数据库连接失败。
- 数据库账号没有建库或建表权限。

处理方式：

1. 确认 MySQL 已启动。
2. 确认 `node/.env` 中数据库配置正确。
3. 重启后端。
4. 再访问 `http://localhost:5174/install` 或 `http://localhost:5174/login`。

### 10.2 前台或后台提示网络请求失败

检查：

- 后端是否正在运行。
- 后端端口是否为 `4000`。
- 前台或后台的 Vite 代理是否正常。
- 是否修改过 `VITE_API_BASE_URL`。
- 生产环境 `CORS_ORIGINS` 是否包含当前访问域名。

### 10.3 登录失败或频繁提示限制

后端对登录接口有限流控制。默认窗口为 15 分钟，登录最多 20 次。可通过以下变量调整：

```env
RATE_LIMIT_WINDOW_MS=900000
LOGIN_RATE_LIMIT_MAX=20
```

### 10.4 忘记后台密码

访问：

```text
http://localhost:5174/recover
```

使用 `node/.env` 中的 `ADMIN_RECOVERY_KEY` 重置密码。该密钥应只由系统管理员保管。

### 10.5 数据库已有旧表或字段缺失

后端启动时会执行 `node/src/bootstrap/initDatabase.js` 中的迁移逻辑，自动补充部分缺失字段。若遇到迁移失败，先备份数据库，再检查后端启动日志中的 SQL 错误。

## 11. 安全注意事项

- 不要提交 `node/.env`。
- 不要把 `JWT_SECRET`、`ADMIN_RECOVERY_KEY`、`DATA_ENCRYPTION_KEY` 发给无关人员。
- 生产环境使用独立数据库账号，避免使用 root。
- 生产环境限制 `CORS_ORIGINS`。
- 管理员密码至少 8 位，并定期更换。
- 数据库上线前先做备份，尤其是在修改表结构或导入数据前。
