# 网站结构与模块说明文档

## 1. 网站整体结构

MaxPlus 网站由三个应用组成：

```text
用户浏览器
├─ 官网前台 web              # 面向访客、买家、客户
├─ 后台管理 sysadmin         # 面向运营、管理员、客服
└─ 后端服务 node             # 统一 API、数据库、登录、加密、日志
```

核心数据流：

```text
后台录入内容
  -> node 后端保存到 MySQL
  -> 官网前台调用 /api/public/bootstrap
  -> 前台把后台内容转换成页面展示数据
```

本项目不是纯静态官网。官网内容、商品、分类、视频、博客、首页模块、订阅弹窗、联系表单、SEO 等都可以通过后台维护。

## 2. 前台网站模块总览

前台目录：

```text
web/src/
├─ main.ts                  # 前台启动入口
├─ App.vue                  # 全站外壳
├─ router/                  # 前台路由
├─ pages/                   # 页面模块
├─ components/              # 页面复用组件
├─ composables/             # 数据、SEO、主题、轮播逻辑
├─ services/                # HTTP 请求、接口地址、安全加密
├─ data/                    # 默认静态数据
├─ directives/              # 动效指令
├─ types/                   # 类型定义
└─ utils/                   # 视频地址等工具函数
```

### 2.1 前台启动模块

文件：

- `web/src/main.ts`
- `web/src/App.vue`

职责：

- 启动 Vue 应用。
- 启动前先调用 `loadPublicData()` 拉取公开数据。
- 注册前台路由。
- 注册 `v-reveal` 滚动进入视口动效指令。
- 引入全局样式 `styles-sporting.css`。

`App.vue` 是全站外壳，所有页面都会包在这里：

```text
SeasonalThemeLayer    # 节日主题背景层
SiteHeader            # 全站导航
RouterView            # 当前页面内容
SiteFooter            # 全站页脚
CookieBanner          # Cookie 弹窗
SubscribeWidget       # 订阅弹窗
BackToTopRocket       # 返回顶部按钮
```

## 3. 前台页面结构

### 3.1 首页 `/`

文件：

- `web/src/pages/HomePage.vue`

首页由多个运营模块拼装：

| 模块 | 组件 | 作用 |
| --- | --- | --- |
| 首屏轮播 | `HeroSlider` | 展示品牌主视觉、首屏文案、行动按钮、开屏动画 |
| 视频展示 | `VideoSection` | 展示后台配置的首页主推视频，并跳转视频列表或详情 |
| 精选商品 | `FeaturedProductsSection` | 展示后台勾选的精选商品 |
| 分类入口 | `CategorySection` | 展示商品分类入口，最多显示 4 个分类卡片 |
| 客户评价 | `ReviewsSection` | 展示评分、评价摘要和评价轮播 |
| 联系表单 | `ContactSection` | 展示首页反馈/咨询表单，提交到后端询盘接口 |

首页模块显隐由后台“首页模块开关”控制。前台读取 `homeContent.sectionToggles`，对应 key：

```text
hero
videos
products
categories
reviews
contact
```

### 3.2 商品列表页 `/products`

文件：

- `web/src/pages/ProductsPage.vue`

作用：

- 展示全部商品。
- 支持按商品分类筛选。
- 支持排序：精选、最新、价格从低到高、价格从高到低。
- 展示分类入口条，点击进入分类详情页。
- 提供搜索入口跳转 `/search`。

数据来源：

- `usePublicData().products`
- `usePublicData().categories`

核心组件：

- `PageBanner`
- `CatalogProductGrid`
- `CatalogProductCard`

### 3.3 商品详情页 `/products/:slug`

文件：

- `web/src/pages/ProductDetailPage.vue`

作用：

- 根据 URL 中的 `slug` 查找商品。
- 展示商品主图、图库缩略图、标题、分类、摘要、价格、购买按钮。
- 商品图片支持鼠标悬停局部放大预览。
- 展示商品描述。
- 展示相关商品。
- 输出 Product 类型结构化数据，利于搜索引擎识别商品。

数据来源：

- `findProductBySlug(slug)`
- `getRelatedProducts(product)`

商品找不到时，会显示“Product Not Found”空状态。

### 3.4 分类详情页 `/categories/:slug`

文件：

- `web/src/pages/CategoryDetailPage.vue`

作用：

- 根据分类 `slug` 展示分类详情。
- 展示分类 Banner、简介、高亮点、统计数据、视觉图形。
- 展示该分类下的商品。
- 支持多维筛选：
  - Sport Type
  - Audience
  - Use
  - Stock
- 支持排序：
  - 默认
  - 最新
  - 价格升序
  - 价格降序
  - Best Selling
- 商品列表分页，每页 4 个。

筛选字段名称来自后台分类配置 `filterConfig`，因此每个分类可以配置不同的筛选文案。

### 3.5 视频列表页 `/videos`

文件：

- `web/src/pages/VideosPage.vue`

作用：

- 展示全部教程视频。
- 支持一级分类和二级分类筛选。
- 显示当前筛选下的视频数量和分类名称。
- 视频卡片展示封面图、分类、主题、标题、摘要。
- 点击进入视频详情页。

数据来源：

- `usePublicData().videoCategories`
- `usePublicData().videos`

视频分类支持父子层级。页面会根据有视频内容的分类自动构建筛选项。

### 3.6 视频详情页 `/videos/:slug`

文件：

- `web/src/pages/VideoDetailPage.vue`

作用：

- 根据视频 `slug` 查找视频。
- 使用 `ExternalVideoPlayer` 播放外部视频。
- 支持 YouTube、Vimeo、Bilibili、iframe 片段和 mp4/webm/ogg/mov/m4v 文件地址。
- 展示视频标题、分类、主题、摘要。
- 展示相关视频。
- 输出 VideoObject 类型结构化数据。

视频地址解析由 `web/src/utils/video.ts` 负责。

### 3.7 博客列表页 `/blog`

文件：

- `web/src/pages/BlogPage.vue`

作用：

- 展示博客文章列表。
- 支持分类筛选。
- 支持关键词搜索。
- 支持最新/最旧排序。
- 支持分页。
- 展示文章数量、分类数量、最近更新时间。
- 页面标题、文案、分页文案来自后台“博客页面”配置。

数据来源：

- `usePublicData().blogs`
- `usePublicData().blogCategories`
- `usePublicData().blogPage`

### 3.8 博客详情页 `/blog/:slug`

文件：

- `web/src/pages/BlogDetailPage.vue`

作用：

- 根据文章 `slug` 查找文章。
- 展示文章封面、分类、标题、摘要、作者、发布时间、预计阅读时间。
- 渲染后台富文本 HTML。
- 自动解析文章中的 `h2`、`h3`，生成右侧目录。
- 展示文章概览信息。
- 展示相关文章。
- 输出 BlogPosting 类型结构化数据。

文章详情页会把文章标题套入后台 SEO 模板 `blogTemplate`。

### 3.9 品牌故事页 `/brand-story`

文件：

- `web/src/pages/AboutPage.vue`

作用：

- 展示品牌主视觉开场动画。
- 展示品牌故事文案段落。
- 展示品牌统计数据。
- 展示品牌亮点。
- 展示商品分类入口。

数据来源：

- `usePublicData().brandStory`
- `usePublicData().categories`

旧路径 `/about` 会重定向到 `/brand-story`。

### 3.10 联系页 `/contact`

文件：

- `web/src/pages/ContactPage.vue`

作用：

- 展示销售邮箱、客服邮箱、电话、社交渠道。
- 展示批发合作表单。
- 展示普通商务咨询表单。
- 表单提交到 `/api/public/inquiry`。

联系信息优先读取后台站点设置；如果后端数据不可用，会使用默认邮箱和默认社交渠道兜底。

### 3.11 购买入口页 `/buy`

文件：

- `web/src/pages/BuyPage.vue`

作用：

- 面向批发/采购流量。
- 展示购买说明点。
- 提供批发询价表单。
- 展示精选商品，方便用户从商品进入采购咨询。

表单提交到 `/api/public/inquiry`，source 为 `wholesale`。

### 3.12 搜索页 `/search`

文件：

- `web/src/pages/SearchPage.vue`

作用：

- 支持通过 URL query `q` 读取搜索词。
- 同时搜索商品、分类、博客文章。
- 无搜索词时展示热门关键词。
- 有结果时分组展示：
  - 分类结果
  - 商品结果
  - 博客结果
- 无结果时展示空状态和热门关键词。

搜索是前端本地搜索，基于已加载的公开数据。

### 3.13 政策页 `/policy/:type`

文件：

- `web/src/pages/PolicyPage.vue`

作用：

- 根据 `type` 从 `web/src/data/catalog.ts` 中查找政策内容。
- 展示隐私、运输、退款、条款等政策文档。
- 政策页目前是静态数据，不走后台接口。

## 5. 前台数据层说明

### 5.1 公开数据加载

核心文件：

- `web/src/composables/usePublicData.ts`
- `web/src/composables/publicData/store.ts`

职责：

- 调用 `/api/public/bootstrap` 拉取公开数据。
- 把后端原始数据转换成前台页面需要的数据结构。
- 维护响应式数据：
  - `siteContent`
  - `brandStory`
  - `categories`
  - `videoCategories`
  - `blogCategories`
  - `products`
  - `videos`
  - `blogs`
  - `blogPage`
  - `seoSettings`
  - `siteSettings`
  - `homeContent`
- 提供查询函数：
  - `findProductBySlug`
  - `findCategoryBySlug`
  - `findVideoBySlug`
  - `findBlogBySlug`
  - `getProductsByCategory`
  - `getRelatedProducts`
  - `getRelatedVideos`
  - `getRelatedBlogs`

如果接口加载失败，前台会保留 `web/src/data/` 中的默认静态数据，避免页面完全空白。

### 5.2 SEO 和结构化数据

核心文件：

- `web/src/composables/useDocumentMeta.ts`
- `web/src/composables/usePageMeta.ts`
- `web/src/composables/useStructuredData.ts`

职责：

- 设置全站默认标题、描述、OG 信息。
- 页面进入时更新当前页面 meta。
- 商品详情输出 Product JSON-LD。
- 视频详情输出 VideoObject JSON-LD。
- 博客详情输出 BlogPosting JSON-LD。

### 5.3 主题与动效

核心文件：

- `web/src/composables/useSiteTheme.ts`
- `web/src/directives/reveal.ts`
- `web/src/composables/useHeroSlider.ts`

职责：

- 根据后台站点主题配置切换 CSS 主题变量。
- 控制滚动进入视口的 reveal 动效。
- 控制首页轮播当前 slide、自动播放、暂停和切换。
- 尊重 `prefers-reduced-motion`，减少动画对用户的影响。

### 5.4 HTTP 与安全请求

核心文件：

- `web/src/services/config.ts`
- `web/src/services/http.ts`
- `web/src/services/security.ts`

职责：

- 统一 API 地址，默认 `/api`。
- 封装 POST 请求。
- 统一处理接口错误和网络错误。
- 对询盘、订阅等敏感提交启用安全加密。

## 14. 新增或修改模块时的维护位置

### 14.1 新增前台页面

需要修改：

- `web/src/pages/` 新增页面组件。
- `web/src/router/index.ts` 注册路由。
- 如需要 SEO，页面内调用 `usePageMeta`。
- 如需要后台数据，优先扩展 `usePublicData`。

### 14.2 新增首页模块

需要修改：

- 前台新增组件到 `web/src/components/`。
- `HomePage.vue` 挂载模块。
- 后台新增配置页面或扩展现有 `HomeContent`。
- 后端 `settingsService.js` 增加保存和归一化字段。
- 数据库 `home_content` 增加 JSON 字段或扩展现有 JSON。

### 14.3 新增后台菜单

需要修改：

- `sysadmin/src/router/index.ts` 新增路由。
- `sysadmin/src/components/layout/SidebarNav.vue` 新增菜单项。
- `sysadmin/src/config/routeMeta.ts` 新增标题和面包屑。
- 如需状态管理，新增或扩展 Pinia store。
- 如需接口，扩展 `node/src/routes/adminRoutes.js` 和对应 service。

### 14.4 新增后端接口

建议流程：

1. 在 `node/src/services/` 写业务逻辑。
2. 在 `node/src/routes/` 注册路由。
3. 在前台或后台 `services/http.ts` 的调用方接入。
4. 如涉及数据库结构，更新 `node/src/bootstrap/initDatabase.js`。
5. 如涉及后台操作，写入 `logs` 操作日志。

## 15. 当前网站模块边界

- 前台负责展示、筛选、搜索、表单提交、SEO 输出。
- 后台负责内容维护、配置维护、线索处理、日志查看。
- 后端负责鉴权、加密、数据库读写、接口聚合、初始化和日志。
- MySQL 负责持久化业务内容和配置。
- `web/src/data/` 只作为默认兜底数据，不是主要运营数据来源。
