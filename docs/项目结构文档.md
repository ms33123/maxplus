# 项目结构文档

## 1. 项目概览

本项目是 MaxPlus 官网与后台管理系统，采用前台、后台、后端分离的三段式结构：

- `web`：官网前台，Vue 3 + Vite + TypeScript。
- `sysadmin`：后台管理端，Vue 3 + Vite + TypeScript + Pinia + Element Plus。
- `node`：后端接口服务，Node.js + Express + MySQL。
- `docs`：项目需求、技术方案、接口数据库说明和本文档。

开发环境中，前台和后台通过 Vite 代理访问后端 `/api`，后端默认监听 `4000` 端口。

## 2. 根目录结构

```text
maxpluswebsite/
├─ docs/                  # 项目文档
├─ web/                   # 官网前台应用
├─ sysadmin/              # 后台管理应用
├─ node/                  # 后端 API 服务
├─ 功能修改.md             # 功能调整记录
├─ 圣诞节主题.md           # 节日主题说明
└─ .gitignore             # Git 忽略配置
```

根目录下的 `node_modules`、构建产物、后端 `.env` 文件不应作为业务代码维护。当前 `.gitignore` 已忽略 `node/.env`、`node/.env.*` 等敏感配置文件。

## 3. 前台应用 `web`

```text
web/
├─ index.html
├─ index-sporting.html
├─ package.json
├─ vite.config.ts
├─ tsconfig.json
├─ styles/                # 拆分后的全局样式
├─ public/
│  └─ images/             # 静态图片资源
└─ src/
   ├─ main.ts             # 前台入口
   ├─ App.vue             # 根组件
   ├─ router/             # 前台路由
   ├─ pages/              # 页面级组件
   ├─ components/         # 前台公共组件
   ├─ composables/        # 数据加载、SEO、主题、轮播等组合函数
   ├─ data/               # 默认静态内容和目录数据
   ├─ directives/         # 自定义指令
   ├─ services/           # API、加密和配置封装
   ├─ types/              # TypeScript 类型定义
   └─ utils/              # 工具函数
```

### 3.1 主要页面路由

前台路由位于 `web/src/router/index.ts`：

| 路径 | 页面 |
| --- | --- |
| `/` | 首页 |
| `/products` | 商品列表 |
| `/products/:slug` | 商品详情 |
| `/categories/:slug` | 分类详情 |
| `/videos` | 视频列表 |
| `/videos/:slug` | 视频详情 |
| `/blog` | 博客列表 |
| `/blog/:slug` | 博客详情 |
| `/brand-story` | 品牌故事 |
| `/contact` | 联系我们 |
| `/buy` | 购买页 |
| `/policy/:type` | 政策页面 |
| `/search` | 搜索页 |

### 3.2 数据来源

前台优先通过 `/api/public/bootstrap` 获取后台配置后的动态内容。接口失败时，部分页面仍可使用 `web/src/data/` 内的默认静态内容作为兜底数据。

主要数据组合函数：

- `web/src/composables/usePublicData.ts`
- `web/src/composables/publicData/store.ts`
- `web/src/composables/usePageMeta.ts`
- `web/src/composables/useStructuredData.ts`

### 3.3 前台配置

`web/vite.config.ts` 默认配置：

- 开发端口：`5173`
- API 代理：`/api` -> `http://127.0.0.1:4000`

前台 API 地址由 `web/src/services/config.ts` 读取：

- 默认：`/api`
- 可通过 `VITE_API_BASE_URL` 覆盖

```text
sysadmin/
├─ index.html
├─ package.json
├─ vite.config.ts
├─ tsconfig.json
└─ src/
   ├─ main.ts             # 后台入口
   ├─ App.vue             # 根组件
   ├─ router/             # 后台路由和鉴权守卫
   ├─ layout/             # 后台整体布局

   ├─ types/              # 后台类型定义
   ├─ utils/              # 本地存储等工具
   └─ data/               # 后台模拟数据
```

### 4.2 登录与状态

后台使用 Pinia 保存登录状态：

- `sysadmin/src/stores/session.ts`：登录、退出、Token、本地存储。
- `sysadmin/src/services/bootstrap.ts`：登录后拉取后台初始化数据。
- `sysadmin/src/stores/catalog.ts`：商品、分类、视频、博客数据。
- `sysadmin/src/stores/settings.ts`：站点和首页配置。
- `sysadmin/src/stores/crm.ts`：询盘、订阅。
- `sysadmin/src/stores/system.ts`：日志和系统信息。

后台所有需要登录的接口会携带 `Authorization: Bearer <token>`。

### 4.3 后台配置

`sysadmin/vite.config.ts` 默认配置：

- 开发端口：`5174`
- API 代理：`/api` -> `http://127.0.0.1:4000`
- 构建时对 `element-plus`、`vue-router`、`pinia` 等依赖做了分包。

后台 API 地址由 `sysadmin/src/services/config.ts` 读取：

- 默认：`/api`
- 可通过 `VITE_API_BASE_URL` 覆盖

## 5. 后端服务 `node`

```text
node/
├─ package.json
├─ package-lock.json
└─ src/
   ├─ server.js           # 服务启动入口
   ├─ app.js              # Express 应用装配
   ├─ bootstrap/          # 运行时准备、数据库初始化
   ├─ config/             # 环境变量、数据库连接、运行状态
   ├─ constants/          # 初始种子数据
   ├─ middleware/         # 鉴权、运行状态保护、错误处理
   ├─ routes/             # 接口路由
   ├─ security/           # RSA/AES 混合加密、密钥生成
   ├─ services/           # 业务服务
   └─ utils/              # 响应、请求解析、错误、JSON、规范化工具
```

### 5.1 后端入口

- `node/src/server.js`：启动服务，调用 `prepareRuntime()` 后监听端口。
- `node/src/app.js`：装配 Express 中间件、CORS、限流、路由和错误处理。
- `node/src/bootstrap/prepareRuntime.js`：检查安装状态，安装完成后初始化数据库。
- `node/src/bootstrap/initDatabase.js`：创建数据库、建表、迁移字段、写入初始数据。

### 5.2 路由结构

所有接口默认挂载在 `/api` 下，前缀由 `API_PREFIX` 控制。

| 路由文件 | 前缀 | 说明 |
| --- | --- | --- |
| `installRoutes.js` | `/api/install` | 安装状态、数据库测试、安装提交 |
| `securityRoutes.js` | `/api/security` | 获取前端加密用公钥 |
| `authRoutes.js` | `/api/auth` | 登录、恢复密钥重置密码 |
| `publicRoutes.js` | `/api/public` | 前台公开数据、询盘、订阅 |
| `adminRoutes.js` | `/api/admin` | 后台管理接口，需要登录 |

后端当前只允许 `POST` 请求方法。前后台通过统一的 `apiPost` 方法调用接口。

### 5.3 主要业务服务

| 服务文件 | 职责 |
| --- | --- |
| `authService.js` | 登录、JWT、密码恢复 |
| `installService.js` | 首次安装、写入 `.env`、创建管理员 |
| `publicService.js` | 前台公开数据聚合 |
| `catalogService.js` | 商品、分类、视频、博客维护 |
| `settingsService.js` | 站点、首页、SEO、订阅弹窗配置 |
| `crmService.js` | 询盘、订阅记录 |
| `subscriptionMailService.js` | 订阅确认邮件 |
| `systemService.js` | 日志、仪表盘统计 |

## 6. 配置文件与环境变量

后端运行配置读取 `node/.env`。该文件包含数据库、JWT、跨域、安全密钥等敏感信息，已被 Git 忽略。

常用变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `HOST` | `0.0.0.0` | 后端监听地址 |
| `PORT` | `4000` | 后端监听端口 |
| `API_PREFIX` | `/api` | API 统一前缀 |
| `CORS_ORIGINS` | `*` | 允许跨域来源，生产环境建议配置具体域名 |
| `MYSQL_HOST` | `127.0.0.1` | MySQL 主机 |
| `MYSQL_PORT` | `3306` | MySQL 端口 |
| `MYSQL_USER` | `root` | MySQL 用户 |
| `MYSQL_PASSWORD` | 空 | MySQL 密码 |
| `MYSQL_DATABASE` | `maxplus_cms` | 数据库名 |
| `APP_INSTALLED` | 空 | 是否完成安装 |
| `JWT_SECRET` | 开发默认值 | JWT 签名密钥 |
| `ADMIN_RECOVERY_KEY` | 开发默认值 | 管理员密码恢复密钥 |
| `DATA_ENCRYPTION_KEY` | 开发默认值 | 询盘、订阅等敏感字段加密密钥 |
| `JWT_EXPIRES_IN` | `12h` | 登录有效期 |
| `RATE_LIMIT_WINDOW_MS` | `900000` | 限流窗口 |
| `RATE_LIMIT_MAX` | `300` | 通用接口限流次数 |
| `LOGIN_RATE_LIMIT_MAX` | `20` | 登录接口限流次数 |

首次安装时，后台安装页会自动生成并写入 `JWT_SECRET`、`ADMIN_RECOVERY_KEY`、`DATA_ENCRYPTION_KEY`。

## 7. 数据流说明

```text
官网 web
  └─ POST /api/public/bootstrap
       └─ node 后端聚合站点设置、首页内容、商品、视频、博客、SEO
            └─ MySQL

后台 sysadmin
  └─ POST /api/auth/login
  └─ POST /api/admin/*
       └─ node 后端校验 JWT 后读写 MySQL

首次安装
  └─ sysadmin /install
       └─ POST /api/install/*
            └─ 写入 node/.env，初始化数据库和管理员账号
```

## 8. 维护约定

- 新增前台页面：在 `web/src/pages/` 添加页面，在 `web/src/router/index.ts` 注册路由。
- 新增前台公共组件：放入 `web/src/components/`。
- 新增后台页面：在 `sysadmin/src/views/` 添加页面，在 `sysadmin/src/router/index.ts` 注册路由，并在 `sysadmin/src/config/routeMeta.ts` 补充标题和面包屑。
- 新增后台状态：优先放入 `sysadmin/src/stores/`，避免页面之间直接传递复杂状态。
- 新增后端接口：在 `node/src/routes/` 添加路由，在 `node/src/services/` 放业务逻辑。
- 修改数据库结构：优先在 `node/src/bootstrap/initDatabase.js` 中补充建表或迁移逻辑。
- 不要提交 `node/.env`、`node_modules/`、构建产物和本地调试文件。
