# Shopify / 独立站首页圣诞主题改版

## 前端任务拆解清单（高级节庆风）

## 一、项目目标

在不大改首页结构的前提下，为独立站首页增加一套 **Christmas / Holiday 高级节庆风 UI**。

目标：

- 保持品牌原有结构和转化路径
- 首页重点模块节日化
- 使用真实感素材，不用纯 CSS 拼图
- PC / 移动端都自然
- 尽量低风险上线，方便回滚

------

# 二、适用范围

适用于以下两种情况：

## 1. Shopify 主题

适合：

- Dawn
- 其他 OS 2.0 主题
- 使用 section / block 结构的主题

## 2. 普通独立站前端

适合：

- HTML + CSS + JS
- Vue / React / Next
- 模板化页面系统

------

# 三、开发原则

## 原则 1：不重构首页

只做以下层面的改造：

- 颜色
- 装饰图层
- 节日 badge
- CTA 节日化
- 活动模块节日化

## 原则 2：装饰与内容分层

- 内容层不改主要逻辑
- 节日元素全部作为 decoration layer
- 不影响点击、SEO、布局流

## 原则 3：可开关

建议整套圣诞主题做成可配置开关：

- `enable_christmas_theme`
- 或 body/class 级别控制：
  - `.theme-christmas`
  - `.holiday-mode`

这样后面下线方便。

------

# 四、开发任务总览

前端任务建议拆成 8 个部分：

1. 全局主题变量
2. 素材目录与资源管理
3. 首页公告条节日化
4. Hero 首屏节日化
5. CTA 按钮戴圣诞帽
6. 商品卡 holiday badge
7. 活动模块 Christmas Specials
8. Footer / 细节装饰 / 响应式 / 性能收尾

------

# 五、详细任务拆解

------

## Task 1：建立节日主题开关

### 目标

保证节日主题可控、可快速回滚。

### 前端任务

- 增加全局 class 或主题配置项
- 所有节日样式都挂在这个 class 下

### 推荐做法

```
<body class="theme-christmas">
```

或 Shopify 里做 setting：

- `Enable Christmas Theme`

### 验收标准

- 去掉开关后页面恢复正常默认样式
- 不影响非节日页面

------

## Task 2：新增节日主题 CSS 变量

### 目标

统一管理圣诞节颜色，避免写死。

### 前端任务

新增主题变量：

```
:root {
  --xmas-burgundy: #8E2430;
  --xmas-forest: #1F5C45;
  --xmas-gold: #C9A86A;
  --xmas-ivory: #FAF7F2;
  --xmas-soft: #F3EEE8;
  --xmas-text: #2E2A27;
  --xmas-white: #FFFFFF;
}
```

### 使用范围

- 公告条
- Hero
- CTA
- badge
- 节日活动模块
- 小范围 icon / 装饰点缀

### 验收标准

- 颜色统一可复用
- 不出现多个近似红色/绿色混乱使用

------

## Task 3：建立节日素材资源目录

### 目标

统一管理真实感素材。

### 前端任务

在项目中新增素材目录，例如：

```
/assets/christmas/
```

### 素材清单

至少包含：

- `christmas-hat.webp`
- `gold-star.webp`
- `velvet-bow.webp`
- `hero-tree.webp`
- `pine-branch.webp`
- `snow-overlay.webp`
- `snow-edge.webp`（可选）
- `holiday-badge.webp`（可选）

### 要求

- 文件命名统一
- 优先 webp
- 透明背景
- 高级、非卡通、非扁平拼接风格

### 验收标准

- 素材都能在本地和线上正常引用
- 移动端不出现尺寸错误或糊图

------

## Task 4：首页顶部公告条节日化

### 目标

让用户第一眼感知 holiday 活动。

### 前端任务

改造首页 announcement bar：

- 改节日配色
- 增加节日文案
- 可选小 icon 点缀

### 建议样式

- 背景：酒红 / 深绿
- 文字：白色 / 金色
- 高度保持原本逻辑

### 可能文案

- Christmas Exclusive: Free Gift Wrap
- Celebrate the Season in Style
- Holiday Gifts Are Here

### Shopify 实施点

如果主题已有 announcement bar section：

- 只扩展 style 和 content setting
- 尽量不改 section 结构

### 验收标准

- PC / 移动端显示正常
- 文案不溢出
- 不影响已有公告逻辑

------

## Task 5：Hero 首屏节日化

### 目标

首页节日氛围主要由 Hero 承担。

### 前端任务

给 Hero section 增加节日 class，比如：

```
<section class="hero hero-xmas">
```

### 具体改动

1. Hero 背景改为暖象牙白渐变
2. 增加一层淡雪花 overlay
3. 增加圣诞树装饰
4. 增加金色星星装饰
5. 增加丝绒蝴蝶结装饰
6. 保持文案和 CTA 为主内容层

### 实现要求

- Hero 容器 `position: relative`
- 节日元素绝对定位
- 所有装饰 `pointer-events: none`
- 不影响主内容排版

### Shopify 实施点

如果 Hero 是 image banner / slideshow / custom hero section：

- 通过 section schema 增加：
  - 是否启用圣诞装饰
  - 圣诞树图片
  - 星星图片
  - 蝴蝶结图片
- 或直接在 section liquid 中写死一版节日结构

### 验收标准

- 桌面端视觉完整
- 移动端素材不压标题
- Hero 不因装饰导致高度异常

------

## Task 6：主 CTA 按钮加圣诞帽

### 目标

强化节日识别点和转化按钮记忆点。

### 前端任务

给首页 Hero 主按钮、活动区主按钮增加节日样式类：

```
<a class="button xmas-btn">
  <span>Shop Christmas</span>
  <img class="xmas-btn-hat" src="/assets/christmas/christmas-hat.webp" alt="">
</a>
```

### 功能要求

- 帽子在按钮左上角
- hover 时轻微摆动
- 帽子不能挡住按钮文字

### Shopify 实施点

如果按钮是主题组件统一输出：

- 增加 modifier class
- 不要全站 button 默认都带帽子
- 只给关键 CTA 开启

### 验收标准

- 帽子位置自然
- hover 动作轻微，不突兀
- 移动端不遮挡点击区

------

## Task 7：商品卡增加 Holiday Badge

### 目标

不改商品主图，低成本提升节日氛围。

### 前端任务

给首页商品卡、精选商品模块、推荐商品模块增加 badge 位。

### 实现方式

在 product card 容器内增加 badge：

```
<div class="product-card">
  <span class="holiday-badge">Holiday Gift</span>
</div>
```

### Badge 文案备选

- Holiday Gift
- Christmas Pick
- Gift Ready
- Limited Holiday Offer

### 样式要求

- 小号胶囊型
- 酒红 / 深绿底
- 白字或淡金字
- 不盖住商品主体

### Shopify 实施点

常见位置：

- `card-product.liquid`
- 产品卡 snippet
- featured collection section
- recommended products section

建议：

- 增加一个可控条件
- 只在首页特定模块启用，避免全站全商品都出现

### 验收标准

- badge 不错位
- 多列商品卡保持统一
- 移动端不压图片主体

------

## Task 8：增加 Christmas Specials 活动区块

### 目标

承接节日活动、礼盒包装、促销信息。

### 前端任务

新增一个首页 section 或复用现有 promotion banner section。

### 模块内容建议

- 标题
- 简短描述
- CTA
- 一张礼物/树/装饰配图
- 可选 2~3 个卖点

### 推荐文案结构

- 标题：Christmas Specials
- 描述：Discover gift-ready picks and limited holiday offers.
- CTA：Explore Holiday Deals

### 样式要求

- 背景象牙白 / 暖米色
- 少量节日装饰图
- CTA 继续使用圣诞帽按钮样式

### Shopify 实施点

建议新建 section：

- `christmas-specials.liquid`

schema 里可配置：

- 标题
- 描述
- 按钮文案
- 按钮链接
- 配图
- 是否启用节日装饰

### 验收标准

- 模块可在首页自由插入排序
- 文案和图片可后台配置
- 不依赖写死内容

------

## Task 9：Footer 顶部加节日边缘装饰

### 目标

补足页面结尾氛围。

### 前端任务

在 footer 顶部加一层轻量装饰：

- 雪边
- 松枝
- 极淡 festive 边框

### 实现要求

- 高度低
- 静态即可
- 不影响 footer 可读性

### Shopify 实施点

- footer section 内插入装饰层
- 或 footer 前插一个 visual divider

### 验收标准

- 不抢 footer 内容
- 移动端不显得拥挤

------

## Task 10：细节动效处理

### 目标

让页面有节日感，但保持高级和克制。

### 前端任务

只做以下轻动效：

- CTA 圣诞帽 hover 微摆
- 金星轻微闪烁
- 个别装饰轻微浮动

### 不要做

- 大面积飘雪
- 粒子库
- 高频闪烁
- 自动旋转
- 复杂 JS 动画

### 验收标准

- 动效不影响性能
- 动效不打断用户浏览
- 动效不廉价

------

## Task 11：移动端适配

### 目标

保持节日感，同时页面不乱。

### 前端任务

对移动端单独处理以下内容：

### 需要保留

- 公告条节日色
- Hero 核心装饰 1~2 个
- CTA 帽子
- 商品卡 badge

### 需要简化

- 蝴蝶结可能隐藏
- 圣诞树缩小
- 雪花 overlay 降低透明度
- 去掉多余装饰图

### 验收标准

- 首屏不拥挤
- 标题和按钮优先级最高
- 装饰不挡文案

------

## Task 12：性能优化

### 目标

增加真实素材后仍保持页面性能可接受。

### 前端任务

- 所有大图压缩为 webp
- 非首屏装饰懒加载
- 避免一次性加载太多大素材
- 不引入重型动画库
- 尽量复用素材

### Shopify 特别注意

- 图片走主题资源或 CDN
- 控制 section 内图片尺寸输出
- 避免 Liquid 输出超大原图

### 验收标准

- 首屏加载无明显变慢
- Lighthouse 不出现大面积图片问题
- 移动端滚动流畅

# 九、测试清单

前端自测至少检查：

## 布局

- 首页 Hero 是否错位
- 商品卡 badge 是否遮挡图片
- 按钮帽子是否压文字
- Footer 装饰是否压内容

## 响应式

- iPhone 常见尺寸
- Android 常见尺寸
- 平板
- 大屏桌面端

## 交互

- CTA 点击正常
- 装饰图不挡点击
- hover 动作自然
- announcement bar 文案不溢出

## 性能

- 首屏图加载是否过慢
- 移动端滚动是否卡顿
- 图片是否糊
- 是否存在未压缩大图

## 回滚

- 去掉 `theme-christmas` 后是否恢复默认
- 关闭 Shopify setting 后是否不显示节日样式

------

# 十、前端最终交付标准

上线前至少满足：

- 首页已有核心模块完成节日化
- 所有节日元素风格统一
- 无明显错位
- 移动端可用
- 性能无明显下降
- 可一键关闭节日主题

------

# 十一、可以直接发给前端的精简版

你可以直接复制这段给前端：

## 前端执行要求

1. 建一个 Christmas theme 开关，不要写死全站
2. 增加节日主题色变量：酒红、深绿、香槟金、象牙白
3. 新建 Christmas 素材目录，统一放真实感 webp 素材
4. 首页公告条改成节日风格
5. Hero 加节日背景和装饰层：树、金星、蝴蝶结、雪花纹理
6. 主 CTA 按钮左上角加圣诞帽，hover 轻微摆动
7. 商品卡增加 Holiday badge，不改商品主图
8. 新增一个 Christmas Specials 首页活动模块
9. Footer 顶部可加淡雪边或松枝装饰
10. 移动端减少装饰数量，保证标题和 CTA 优先
11. 所有装饰图必须不挡点击、不过度动画
12. 保证节日主题可快速回滚
